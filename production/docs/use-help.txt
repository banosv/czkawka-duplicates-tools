CZKAWKA DUPLICATE FILE TOOLS - QUICK USAGE GUIDE
=================================================

OVERVIEW
--------
Find and analyze duplicate files in 4 simple steps:
1. Scan for duplicates → 2. Convert to table → 3. Import to spreadsheet → 4. Analyze

STEP 1: SCAN FOR DUPLICATES
---------------------------
./czkawkaDupFind.sh ~/Documents duplicates
(Scans Documents folder, saves as duplicates.json)

STEP 2: CONVERT TO CSV TABLE
----------------------------
./czkawka_to_table.sh duplicates.json ~/Documents duplicates.csv
(Focuses on Documents folder, creates duplicates.csv with one row per duplicate)

STEP 3: IMPORT TO SPREADSHEET
-----------------------------
1. Open: libreoffice --calc duplicates.ods
2. Copy entire CSV contents (Ctrl+A, Ctrl+C)
3. Paste directly into 'duplicates' sheet at A1 (Ctrl+V)
4. Use filters and sorting to analyze data

STEP 4: ANALYZE AND MANAGE FILES
--------------------------------
• Use filters to sort by size, folder, or file type
• Click on folder/filename cells and press F3 to open in file manager
• Each row shows one duplicate relationship (original → duplicate)
• Use decision marking shortcuts to efficiently process duplicates

FILE MANAGER INTEGRATION SETUP
------------------------------
1. Tools → Customize → Keyboard
2. Assign F3 to macro: OpenInFileManagerFromActiveCell
3. Enable macros: Tools → Options → Security → Macro Security (Medium)

COLUMN LAYOUT (ONE ROW PER DUPLICATE)
-------------------------------------
A: Original folder path    B: Original filename
C: Duplicate folder path   D: Duplicate filename
E: File size               F: Hash value
G: Keep_Original          H: Keep_Duplicate          I: Action

Note: Files with multiple duplicates appear in multiple rows

DECISION MARKING SHORTCUTS
--------------------------
F3: Open both file locations in file manager
Ctrl+3: Keep original, delete duplicate (green/red colors)
Ctrl+4: Keep duplicate, delete original (red/green colors)  
Ctrl+6: Mark files for review (yellow color)
Ctrl+7: Show decision statistics and progress
Ctrl+8: Clear decisions for selected rows
Ctrl+9: Apply AutoFilter for decision analysis

QUICK COMMANDS
--------------
Scan home directory:       ./czkawkaDupFind.sh ~ home_scan
Convert with focus:        ./czkawka_to_table.sh home_scan.json ~/Documents docs.csv
Open spreadsheet:          libreoffice --calc duplicates.ods

COMMON SCANNING TARGETS
-----------------------
Downloads cleanup:         ./czkawkaDupFind.sh ~/Downloads downloads_scan
Pictures analysis:         ./czkawkaDupFind.sh ~/Pictures photos_scan
Cloud sync check:          ./czkawkaDupFind.sh ~/MEGAsync cloud_scan
Full home scan:            ./czkawkaDupFind.sh ~ complete_scan

UNDERSTANDING RESULTS
--------------------
Each CSV row represents one duplicate relationship:
- Original file in your focus folder → One duplicate location
- Files with 3 duplicates = 3 separate rows
- Better for filtering and analysis than combined rows

EFFICIENT WORKFLOW
------------------
1. Sort by file size (Column E) - handle large duplicates first
2. For each row: F3 (view files) → Compare → Ctrl+3/Ctrl+4 (decide) → auto-advance
3. Use Ctrl+6 for complex cases needing review
4. Periodically use Ctrl+7 to check progress
5. Use Ctrl+9 to filter by decision types for final review

COLOR CODING SYSTEM
-------------------
Green cells: Files marked to keep
Red cells: Files marked for deletion
Yellow cells: Files needing review
No color: Unprocessed files

SUPPORTED FILE MANAGERS
-----------------------
• Krusader: Dual-pane view for easy comparison
• Nemo: Tabbed interface with file selection
• System default: Fallback for other file managers
Install: sudo apt install nemo krusader

TROUBLESHOOTING
---------------
Scripts not executable:    chmod +x *.sh
Missing dependencies:      sudo apt install jq nemo krusader
Macro not working:         Enable macros in LibreOffice security settings
Large scans slow:          Use -f flag for text format: ./czkawkaDupFind.sh ~/path output -f

STATISTICS AND PROGRESS
----------------------
Ctrl+7 shows comprehensive statistics:
- Files marked to keep original vs duplicate
- Files needing review
- Completion percentage
- Total duplicate relationships processed
